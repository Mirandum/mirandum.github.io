Arquitectura MVC

La aplicación está sobre el lenguaje de programación Ruby, para hacer más sencilla la implementación usamos Rails que es un framework para hacer aplicaciones web con Ruby.

Rails viene por defecto con arquitectura MVC es decir , que cuando creamos una aplicación con rails, se creará una carpeta llamada “app” donde se encontrarán los subdirectorios “model”, “view” y “controller”.

En la carpeta “model” se encuentran las entidades del negocio como lo son compra, producto , consumidor y administrador; en la carpeta “view” tenemos todos los archivos erb.html que son las vistas del usuario, es decir donde se despliega toda la información sobre el modelo actual y en la carpeta “controller” se encuentran los controladores que permiten la comunicación entre la vista y el modelo con el fin de tener una aplicación dinámica.

Diagrama: Flujo MVC para ver productos

Diagrama

Veamos MVC en acción.

El flujo para ver los productos en la aplicación sería el siguiente:

  • 1. El usuario hace una petición al navegador para la URL /productos
  • 2. El Rails router mapea la URL a la acción del controlador correspondiente, en este caso la acción es “index” en el controlador “productos”.
  • 3. La acción index le pide al modelo Producto todos los productos que hay. (Productos.all)
  • 4. El modelo Producto recupera todos los productos de la base de datos.
  • 5. El modelo Producto retorna la lista de productos al controlador “productos”.
  • 6. El controlador captura la lista de productos en la variable @productos la cual pasa a la vista index.
  • 7. La vista index corre la página como un HTML.
  • 8. El controlador “productos” envía la página HTML al navegador.
Cómo pudimos darnos cuenta la vista y el modelo nunca están en contacto directamente, el controlador es el que hace posible la concordancia entre éstas.

NOTA: Las vistas en Ruby estan en formato .erb.html o html.erb. erb: Embedded RuBy. Lo cual nos dice que es un archivo html con código Ruby embebido.
